# Bronly — Универсальная SaaS-платформа бронирования

Монорепо с бэкендом (Fastify + PostgreSQL) и фронтендом (Next.js 16).  
Работает как маркетплейс (клиент ищет заведение) и как white-label инструмент (бизнес встраивает на сайт).

## Стек технологий

| Слой | Технология |
|------|-----------|
| Бэкенд | Node.js, TypeScript, **Fastify v5** |
| ORM | **Prisma v5**, PostgreSQL |
| Аутентификация | Имя + телефон, подтверждение через Telegram-бота или SMS-код (без пароля). Сессия — httpOnly-куки: access-токен (2 часа) + refresh-токен (до ~1 года, скользящий) |
| Валидация | **Zod** |
| Фронтенд | **Next.js 16** App Router, React |
| Стили | **Tailwind CSS v4** |
| Email | **Resend SDK** (graceful fallback без API-ключа) |
| SMS | Провайдеро-независимая обёртка (`lib/sms.ts`, Twilio/webhook/лог-в-консоль) |
| Платежи | **Finik (Averspay)** — RSA-ключ на бизнес, вебхук-авторитетный. На момент написания отключены (`FINIK_ENABLED=false`), провайдер ещё не выбран/подключён — все брони проходят как обычно, просто без онлайн-оплаты |
| Монорепо | **pnpm workspaces** |

## Архитектура

```
booking_system/
├── apps/
│   ├── api/          # Fastify REST API
│   │   ├── src/
│   │   │   ├── routes/      # auth, businesses, bookings, resources, services,
│   │   │   │                # staff, stats, payment, clients, reviews, upload,
│   │   │   │                # admin, telegram, reminder-rule
│   │   │   ├── lib/         # prisma, email, sms, telegram, finik, datetime
│   │   │   └── plugins/     # authenticate decorator
│   │   └── prisma/
│   │       └── schema.prisma
│   └── web/          # Next.js фронтенд
│       └── src/
│           ├── app/         # страницы (App Router)
│           ├── components/  # UI компоненты
│           ├── context/     # AuthContext
│           └── lib/         # api.ts (HTTP клиент), businessTypes.ts
└── packages/         # shared types (будущее)
```

**Мультитенантность:** каждый `Business` — это тенант. Все `Resource`, `Booking`, `Service`, `StaffMember` привязаны к `businessId`.

## Функциональность

### Для клиентов (покупателей)

- **Маркетплейс** — поиск заведений по названию, адресу, типу бизнеса; пагинация; дебаунс поиска
- **Страница заведения** — описание, расписание мастеров/номеров, список услуг
- **Бронирование** — адаптивный мастер под тип бизнеса; inline-авторизация если не залогинен
- **Оплата (депозит)** — фиксированная предоплата за услугу/ресурс, если её настроил бизнес; остаток — на месте. Пока не выбран платёжный провайдер, онлайн-оплата отвечает понятной ошибкой вместо фейкового успеха
- **Профиль / мои брони** — история бронирований, отмена с учётом политики отмены бизнеса
- **Email-уведомления** — подтверждение и отмена бронирования
- **Telegram-напоминания** — настраиваемые бизнесом правила («за неделю», «за день» и т.д.)

### Для владельцев бизнеса

- **Дашборд** — статистика за день/месяц, доход, ближайшие бронирования
- **Управление бронями** — таблица/календарь с фильтрами по статусу и дате, смена статуса, ручное бронирование и блокировка слотов
- **Клиенты** — сводный список всех, с кем бизнес взаимодействовал (онлайн-клиенты и ручные записи, объединённые по телефону)
- **Ресурсы** — CRUD номеров/мастеров/столиков, настройка расписания со слотами, дни-off/особые часы
- **Услуги** — CRUD услуг с ценой, продолжительностью и депозитом
- **Персонал** — добавление сотрудников по телефону, назначение должностей и ролей
- **Отзывы** — просмотр и ответ на отзывы клиентов
- **Политика отмены** — свободная отмена / штраф / без возврата, по часам до начала брони
- **Настройки** — создание новых бизнесов, фото, ссылка на страницу бронирования, подключение платёжного провайдера

### Безопасность

- Сессия — httpOnly-куки (access + refresh), не localStorage; refresh ротируется при каждом использовании
- Rate limiting на аутентификации и создании ресурсов
- Проверка владения (`business.ownerId`) во всех защищённых маршрутах, привязанных к бизнесу
- Цена берётся из БД — клиент не может передать свою сумму
- Вебхук платёжного провайдера проверяется по подписи, обрабатывается идемпотентно
- Бронировать прошедшие даты нельзя

## API эндпоинты

> `[AUTH]` — требует сессионные куки (access-токен), выставляемые после логина

### Auth — `/api/auth`

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/start` | Начать вход: имя + телефон → challengeId |
| POST | `/challenge/:id/send` | Отправить код (Telegram или SMS) |
| GET | `/challenge/:id/status` | Статус подтверждения (для поллинга deep-link'а) |
| POST | `/challenge/:id/verify` | Подтвердить код → создаёт сессию (cookies) |
| POST | `/refresh` | Ротировать сессию по refresh-куке |
| POST | `/logout` | Отозвать сессию, очистить куки |
| GET | `/me` | `[AUTH]` Профиль текущего пользователя |
| PATCH | `/me` | `[AUTH]` Обновление имени |

### Businesses — `/api/businesses`

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Список активных бизнесов (query, type, page, limit) |

| GET | `/:slug` | Публичная страница бизнеса |
| POST | `/` | `[AUTH]` Создать бизнес |

## Установка и запуск

### Требования

- Node.js 20+
- pnpm 9+
- PostgreSQL 15+

### Шаги

```bash
# 1. Клонировать и установить зависимости
git clone <repo>
cd booking_system
pnpm install

# 2. Настроить env
cp apps/api/.env.example apps/api/.env
# Заполнить DATABASE_URL, JWT_SECRET, FRONTEND_URL

cp apps/web/.env.example apps/web/.env.local
# Заполнить NEXT_PUBLIC_API_URL

# 3. Применить миграции БД
cd apps/api
pnpm db:migrate     # Применить миграции
pnpm db:generate    # Сгенерировать Prisma Client

# 4. Запуск в dev-режиме (оба приложения параллельно)
cd ../..
pnpm dev
```

После запуска:
- API: http://localhost:4000
- Web: http://localhost:3000
- Prisma Studio: `cd apps/api && pnpm db:studio` → http://localhost:5555

## Переменные окружения

### `apps/api/.env`

| Переменная | Обязательная | Описание |
|-----------|-------------|----------|
| `DATABASE_URL` | ✅ | PostgreSQL connection string |
| `JWT_SECRET` | ✅ prod | Секрет для подписи access-токена |
| `FRONTEND_URL` | ✅ prod | URL фронтенда (CORS whitelist, редиректы после оплаты) |
| `PORT` | ❌ | Порт API (по умолчанию 4000) |
| `RESEND_API_KEY` | ❌ | API ключ Resend для email. Без него email пишутся только в лог |
| `ALLOW_TEST_PHONES` | ❌ | `true` включает 20 тестовых номеров с фиксированным кодом (QA). Никогда не включать в реальном проде |
| `TELEGRAM_BOT_TOKEN`, `TELEGRAM_BOT_USERNAME`, `TELEGRAM_WEBHOOK_SECRET` | ❌ | Бот для входа/напоминаний. Без токена — лог вместо отправки |
| `SMS_PROVIDER`, `SMS_WEBHOOK_URL`, `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_FROM` | ❌ | Провайдер SMS-кодов. Без настройки — лог вместо отправки |
| `ENCRYPTION_KEY` | ❌ | 32 байта в base64 — шифрование приватного ключа Finik в БД (`openssl rand -base64 32`) |
| `FINIK_API_URL`, `FINIK_WEBHOOK_PUBLIC_KEY` | ❌ | Платёжный провайдер Finik. Интеграция готова, но выключена (`FINIK_ENABLED=false` в коде) до выбора провайдера |
| `API_PUBLIC_URL` | ❌ | Публичный URL API — используется для формирования webhookUrl платёжного провайдера |

### `apps/web/.env.local`

| Переменная | Описание |
|-----------|----------|
| `NEXT_PUBLIC_API_URL` | URL бэкенда (доступен в браузере) |

## Деплой

### Рекомендуемый стек

- **API**: Railway / Render / VPS (pm2 + node)
- **Web**: Vercel / Netlify
- **БД**: Railway PostgreSQL / Supabase / Neon

### Продакшн-чеклист

- [ ] `NODE_ENV=production` в обоих сервисах
- [ ] `JWT_SECRET` — случайная строка 64 символа (`openssl rand -hex 64`)
- [ ] `FRONTEND_URL` — реальный домен фронтенда
- [ ] `DATABASE_URL` — продакшн БД (SSL)
- [ ] `RESEND_API_KEY` — API ключ для email
- [ ] `ALLOW_TEST_PHONES` не установлена (или явно `false`)
- [ ] Выбрать и подключить платёжного провайдера, включить `FINIK_ENABLED` (или эквивалент) в коде
- [ ] Запустить `pnpm db:migrate` на продакшн БД
- [ ] HTTPS на обоих доменах

## Структура БД (ключевые модели)

```
User ──────────────────────── owns ──► Business
                                          │
                              has ──────► Resource
                                          │
                              has ──────► Schedule (дни, время, длина слота)
                                          │
                              has ──────► ScheduleException (закрыт, изменённое время)

User ──── creates ──────────────────────► Booking ◄── uses ── Resource
                                          │                    Service
                                          │
                                          └──── has ──────────► Payment

Business ──── has ──────────────────────► Service
Business ──── has ──────────────────────► StaffMember ◄── is ── User
```
