# Гайд по функционалу — пошаговый обход

Всё что можно потрогать в системе, по шагам. Оба сервера должны быть запущены.

```
API:  http://localhost:4000
Web:  http://localhost:3000
```

Email:    demo@booking.kg

## БЛОК 1 — Маркетплейс (взгляд клиента)

### 1.1 Главная страница

**URL:** http://localhost:3000

- Видишь hero с кнопкой "Найти заведение"
- Внизу — 6 категорий: Отели, Рестораны, Салоны, Коворкинг, Спорт, Медицина
- Нажми на любую категорию → попадёшь в каталог с фильтром по типу
- Навбар: если не залогинен — кнопки "Войти" + "Бизнесу", если залогинен — имя пользователя + "Дашборд"

### 1.2 Каталог заведений

**URL:** http://localhost:3000/explore

**Что проверить:**
1. Введи в поиск `Salon` — список фильтруется через API с debounce 350мс
2. Нажми фильтр `Салоны` — только салоны
3. Нажми фильтр `Отели` — только отели
4. Нажми `Все` — сброс фильтра
5. Нажми на карточку заведения → страница бронирования

## БЛОК 2 — Бронирование (адаптивный флоу)

### 2.1 Салон — флоу: услуга → мастер → дата → слот → форма

**URL:** http://localhost:3000/b/demo-salon

Шаги:
1. Открой страницу — видишь описание салона, список мастеров и услуг
2. Нажми **"Забронировать"**
3. **Шаг "Услуга"** — выбери из списка (Стрижка 800 сом / Маникюр 600 сом / Укладка 400 сом)
4. **Шаг "Мастер"** — выбери Мастер Айгуль
5. **Шаг "Дата"** — выбери любой будний день (пн-сб)
6. **Шаг "Время"** — появятся слоты 09:00–19:00 по 1 часу
7. **Шаг "Форма"** — если не залогинен: появятся вкладки "Войти" / "Регистрация"
   - Зарегистрируйся прямо в форме бронирования
   - Если уже залогинен: вместо формы — зелёный блок с именем и email
8. Нажми **"Забронировать"** → попадёшь на страницу подтверждения
9. Видишь номер брони, статус "Ожидает подтверждения"

> ⚡ Попробуй выбрать тот же слот снова — получишь ошибку "Это время уже занято"

### 2.2 Отель — флоу: номер → диапазон дат → гости → форма

**URL:** http://localhost:3000/b/super-hotel

Шаги:
1. Нажми **"Забронировать"**
2. **Шаг "Номер"** — выбери Президентский люкс
3. **Шаг "Даты"** — кликни дату заезда, потом дату выезда (визуальный range picker)
4. **Шаг "Гости"** — нажимай +/− счётчик
5. **Шаг "Форма"** — заполни или войди
6. Подтверждение с количеством ночей

### 2.3 Оплата (dev-мок Bakai)

После бронирования с услугой на странице подтверждения появляется кнопка **"Оплатить N сом"**.

1. Нажми кнопку оплаты
2. Попадёшь на мок-страницу Bakai: http://localhost:3000/booking/pay-mock
3. Видишь сумму и номер транзакции
4. Нажми **"Оплатить"** → редирект на `/booking/success`
5. Или **"Отменить"** → редирект на `/booking/failed`

> В prod режиме вместо мок-страницы — настоящий Bakai PayLink

## БЛОК 3 — Аккаунт клиента

### 3.1 Регистрация

**URL:** http://localhost:3000/register

Заполни: Имя, Email, Телефон (необязательно), Пароль (мин. 6 символов)

> Попробуй зарегистрироваться с тем же email — получишь "Этот email уже зарегистрирован"

### 3.2 Вход

**URL:** http://localhost:3000/login

> Введи неверный пароль — получишь "Неверный email или пароль" (не раскрывает, что именно неверно)

### 3.3 Профиль клиента

**URL:** http://localhost:3000/profile *(нужна авторизация)*

**Вкладка "Брони":**
- Список твоих бронирований с цветными статусами
- Кнопка "Отменить" у активных броней
- Нажми отмену → статус становится "Отменена"

**Вкладка "Данные":**
- Измени имя и телефон
- Нажми "Сохранить" → данные обновятся и в навбаре

**Вкладка "Пароль":**
- Введи текущий пароль + новый (мин. 6 символов)
- Неверный текущий → ошибка "Неверный текущий пароль"

## БЛОК 4 — Дашборд владельца

**Войди как:** `demo@booking.kg` / `demo1234`

### 4.1 Обзор

**URL:** http://localhost:3000/dashboard

- 4 карточки: Брони сегодня / За месяц / Всего / Ожидают
- Выручка за месяц (считается по оплаченным бронированиям)
- Таблица "Ближайшие брони" (следующие 5)
- Быстрые ссылки: Все брони, Ресурсы, Услуги, Персонал

> На мобильном — нажми кнопку ☰ в левом верхнем углу → откроется боковое меню

### 4.2 Управление бронями

**URL:** http://localhost:3000/dashboard/bookings

- Фильтры по статусу (Все / Ожидает / Подтверждена / Завершена / Отменена)
- Поиск по дате
- Кнопки смены статуса на каждой карточке: Подтвердить / Завершить / Отменить

### 4.3 Ресурсы

**URL:** http://localhost:3000/dashboard/resources

**Что сделать:**
1. Нажми **"+ Добавить ресурс"** → модал с формой
2. Заполни: Название (например "Мастер Зарина"), Вместимость 1
3. Нажми "Создать" — ресурс появится в списке
4. Нажми **"Расписание"** рядом с ресурсом → форма добавления расписания
5. Выбери дни недели, время начала/конца, длину слота
6. Сохрани → теперь клиенты увидят слоты для этого мастера

> Закрой модал клавишей **Escape**

### 4.4 Услуги

**URL:** http://localhost:3000/dashboard/services

**Что сделать:**
1. Нажми **"+ Добавить"** → модал
2. Заполни: Название, Описание, Длительность (мин), Цена (сом)
3. Создай → услуга появится в списке с бейджами длительности и цены
4. Нажми **"Изм."** → отредактируй название
5. Нажми **"Удал."** → подтверждение → услуга удалена

### 4.5 Персонал

**URL:** http://localhost:3000/dashboard/staff

**Что сделать:**
1. Нажми **"+ Добавить"**
2. Введи email зарегистрированного пользователя (например `ruslan@gmail.com`)
3. Укажи должность (например "Администратор")
4. Нажми "Добавить" → сотрудник появится в списке

> Попробуй добавить несуществующий email → ошибка "Пользователь не найден"

### 4.6 Настройки

**URL:** http://localhost:3000/dashboard/settings

- Список твоих бизнесов с типом и slug
- Ссылка на публичную страницу бронирования
- Кнопка **"+ Новый бизнес"** → форма создания

**Создай тестовый бизнес:**
1. Название: `Мой Ресторан`
2. Slug: `my-restaurant-test` (только латиница, дефис)
3. Тип: Ресторан
4. Адрес: любой
5. "Создать" → бизнес появился в списке

> Попробуй тот же slug снова → "Slug already taken"

## БЛОК 5 — Граничные случаи (безопасность)

### 5.1 Rate limiting

Открой DevTools → Console и выполни:

```javascript
// 11 быстрых логинов с неверным паролем
for(let i=0;i<11;i++){
  fetch('/api/auth/login',{method:'POST',headers:{'Content-Type':'application/json'},
  body:JSON.stringify({email:'test@test.com',password:'wrong'})})
  .then(r=>r.json()).then(d=>console.log(i+1, r.status, d))
}
```

На 11-й попытке придёт `429: "Слишком много попыток. Подождите минуту."`

### 5.2 Конфликт бронирования

1. Запомни время и дату только что созданной брони
2. Попробуй забронировать то же время у того же мастера
3. Получишь ошибку **"Это время уже занято"** на шаге формы

### 5.3 Авто-логаут при истечении токена

Можно проверить через DevTools:
```javascript
localStorage.setItem('token', 'invalid.token.here')
location.reload()
```
Токен невалиден → `GET /api/auth/me` вернёт 401 → приложение разлогинит и покажет кнопку "Войти"

### 5.4 Защита данных

Попробуй в браузере (будучи залогиненным):
```
GET http://localhost:4000/api/bookings/any-booking-id-here
```
Если бронь чужая → `403 Forbidden`

## БЛОК 6 — Email уведомления

При создании брони система пытается отправить:
- **Клиенту:** "Ваша бронь подтверждена" с деталями
- **Владельцу:** "Новая бронь" с данными клиента

В dev-режиме без `RESEND_API_KEY` — письма только логируются в консоль API.  
В prod-режиме с API ключом — отправляются реальные письма.

## БЛОК 7 — API напрямую (Postman / curl)

### Быстрый тест всего потока через curl

```bash
API=http://localhost:4000

# 1. Регистрация
TOKEN=$(curl -s -X POST $API/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"me@test.com","password":"pass123","name":"Test"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")

echo "Token: ${TOKEN:0:30}..."

# 2. Получить список заведений
curl -s "$API/api/businesses?limit=3" | python3 -c "
import sys,json; d=json.load(sys.stdin)
for b in d['businesses']: print(b['name'], '->', b['slug'])"

# 3. Слоты конкретного мастера
NEXT_MON=$(python3 -c "
from datetime import date,timedelta
t=date.today(); d=(0-t.weekday())%7 or 7
print(t+timedelta(days=d))")

# Узнать resource ID через публичный эндпоинт
RES_ID=$(curl -s "$API/api/businesses/demo-salon" | \
  python3 -c "import sys,json; print(json.load(sys.stdin)['resources'][0]['id'])")

curl -s "$API/api/resources/$RES_ID/slots?date=$NEXT_MON" | \
  python3 -c "import sys,json; d=json.load(sys.stdin); print(len(d['slots']),'слотов на',$NEXT_MON)"
```
