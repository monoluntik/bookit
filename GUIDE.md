# Bookit — Полный гайд по платформе

> Универсальная SaaS-платформа для онлайн-бронирования. Три роли: клиент, владелец бизнеса, суперадмин.

---

## Роли

| Роль | Как получить |
|---|---|
| **Клиент** | Дефолт при регистрации |
| **Владелец бизнеса** | Автоматически при создании первого бизнеса |
| **Суперадмин** | Seed-аккаунт: `admin@booking.local` / `Admin1234!` |

---

## Часть 1 — Клиент

### Регистрация и вход

| Страница | Описание |
|---|---|
| `/register` | Создать аккаунт — имя, email, пароль |
| `/login` | Войти в аккаунт |
| `/forgot-password` | Запросить сброс пароля на email |
| `/reset-password` | Установить новый пароль по ссылке из письма |

---

### Поиск бизнесов — `/explore`

- Карточки всех активных бизнесов
- Фильтры: по типу (салон, отель, ресторан, коворкинг, спорт, медицина), по городу
- Сортировка: по рейтингу, по новизне
- Поиск по названию
- Каждая карточка показывает: название, тип, адрес, рейтинг, минимальную цену

---

### Страница бизнеса — `/b/[slug]`

- Название, описание, адрес, телефон, email
- Галерея фотографий
- Список услуг с ценами и длительностью
- Список мастеров / ресурсов с фото
- Кнопка **"Забронировать"** — запускает форму бронирования
- Отзывы клиентов с оценками и ответами владельца

---

### Бронирование

Форма адаптируется под тип бизнеса:

**Салон / Медицина** (`serviceFirst`):
1. Выбор услуги
2. Выбор мастера
3. Выбор даты
4. Выбор временного слота
5. Подтверждение и оплата

**Отель** (`dateRange`):
1. Выбор номера
2. Выбор диапазона дат (заезд → выезд)
3. Количество гостей
4. Подтверждение и оплата

**Ресторан / Коворкинг / Спорт** (`timeSlot` + `guestCount`):
1. Выбор ресурса (стол / место / корт)
2. Дата и временной слот
3. Количество гостей
4. Подтверждение и оплата

**После бронирования:**
- Мок-страница оплаты `/booking/pay-mock`
- Кнопка **"Оплатить"** → статус `CONFIRMED`, редирект на `/booking/success`
- Кнопка **"Отказать"** → статус `FAILED`, редирект на `/booking/failed`

---

### Мои брони — `/my-bookings`

- Список всех броней с фильтром по статусу
- Статусы: `Ожидает` / `Подтверждена` / `Завершена` / `Отменена` / `Неявка`
- Кнопка **"Отменить"** — доступна если бизнес разрешает и не истёк дедлайн
- После завершённой брони — форма отзыва с оценкой ★ и комментарием

---

### Профиль — `/profile`

- Изменить имя и телефон
- Сменить пароль
- Просмотр истории активности

---

## Часть 2 — Владелец бизнеса

После регистрации перейди в `/dashboard/settings` → нажми **"Создать бизнес"**.
Роль автоматически изменится на `BUSINESS_OWNER`.

---

### Дашборд — `/dashboard`

Стартовый экран владельца:
- Ближайшие брони на сегодня и завтра
- Количество броней в статусе "Ожидает подтверждения"
- Быстрые ссылки на все разделы

---

### Брони — `/dashboard/bookings`

- Полный список броней по всем бизнесам
- Фильтры по статусу, дате, бизнесу
- Действия с каждой бронью:
  - **Подтвердить** — перевести в `CONFIRMED`
  - **Завершить** — перевести в `COMPLETED`
  - **Отменить** — перевести в `CANCELLED`
- Детали: клиент, услуга, ресурс, время, комментарий, сумма

---

### Ресурсы — `/dashboard/resources`

Ресурс = то, что бронируют (стол, мастер, номер, корт, кабинет).

**Создание ресурса:**
- Название и описание
- Вместимость (количество гостей)
- Базовая цена
- Фото

**Расписание ресурса:**
- Рабочие часы для каждого дня недели
- Отметить день как выходной (`isClosed`)
- Исключения: кастомные часы на конкретную дату

---

### Услуги — `/dashboard/services`

- Создать услугу: название, цена, длительность в минутах, описание
- Услуги отображаются на публичной странице бизнеса
- Клиент выбирает услугу при бронировании (для типов `serviceFirst`)

---

### Персонал — `/dashboard/staff`

- Добавить сотрудника: имя, должность, фото
- Привязать к ресурсу (мастер → рабочее место)
- Персонал отображается на странице бизнеса

---

### Выходные — `/dashboard/schedule`

- Добавить конкретные даты как выходные / праздники для всего бизнеса
- В эти дни слоты недоступны для бронирования у всех ресурсов

---

### Политика отмены — `/dashboard/cancellation`

- Разрешить или запретить клиентам самостоятельно отменять брони
- Дедлайн отмены: за сколько часов до начала можно отменить
- Штраф за позднюю отмену (в процентах от суммы)

---

### Статистика — `/dashboard/stats`

- **KPI-карточки:** выручка, количество броней, средний чек, новые клиенты — с % изменением к прошлому месяцу
- **График броней** за последние 30 дней (SVG bar chart)
- **Donut chart** — разбивка броней по статусам
- **Популярность услуг** — горизонтальные бары
- **Ближайшие брони** — список на следующие дни

---

### Отзывы — `/dashboard/reviews`

- Средний рейтинг и распределение по звёздам (1–5)
- Список всех отзывов с именем клиента, датой, оценкой, комментарием
- **Ответить на отзыв** — ответ виден клиенту на странице бизнеса
- Редактировать существующий ответ

---

### Настройки — `/dashboard/settings`

- Редактировать данные бизнеса: название, тип, описание, адрес, телефон, email
- Загрузить логотип
- Добавить фото в галерею
- Создать дополнительный бизнес (один аккаунт — несколько бизнесов)

---

## Часть 3 — Суперадмин

Войди как `admin@booking.local` / `Admin1234!`.
В дашборде появится кнопка **"🛡️ Админка"** в боковом меню.

---

### Обзор — `/admin`

- Всего пользователей (+ прирост за месяц)
- Активных бизнесов (из общего числа)
- Всего броней (+ за месяц)
- Общая выручка платформы
- График активности за 30 дней
- Распределение бизнесов по категориям
- Последние 5 броней по всей платформе

---

### Пользователи — `/admin/users`

- Поиск по имени или email
- Фильтр по роли: клиент / владелец / суперадмин
- **Изменить роль** прямо в строке таблицы
- **Заблокировать** — пользователь не сможет войти
- **Разблокировать** — восстановить доступ
- Количество броней и бизнесов у каждого пользователя

---

### Бизнесы — `/admin/businesses`

- Поиск по названию или slug
- Фильтр по типу и статусу активности
- **Открыть ↗** — перейти на публичную страницу
- **Скрыть** — бизнес исчезает из каталога `/explore`
- **Активировать** — вернуть в каталог
- Статистика: брони, отзывы, ресурсы

---

### Отзывы — `/admin/reviews`

- Все отзывы по всей платформе
- Ссылка на бизнес для контекста
- **Удалить** — модерация спама и нарушений

---

## Локальный запуск

```bash
# Из корня проекта
pnpm --filter @booking/api dev     # API на :4000
pnpm --filter @booking/web dev     # Фронт на :3000
```

Переменные окружения:
- `apps/api/.env` — `DATABASE_URL`, `JWT_SECRET`, `FRONTEND_URL`
- `apps/web/.env.local` — `NEXT_PUBLIC_API_URL`
